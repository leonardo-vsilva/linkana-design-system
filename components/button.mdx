| Variante | Ênfase | Uso |
|----------|--------|-----|
| `default` | Alta | Ação principal do fluxo — salvar, confirmar, publicar |
| `secondary` | Média | Ação alternativa válida — salvar rascunho, duplicar, visualizar |
| `destructive` | Alta | Ação irreversível — deletar, remover |
| `outline` | Baixa | Ação de saída ou desfazer em grupos de ações — cancelar, voltar, limpar |
| `ghost` | Mínima | Ação contextual discreta — editar inline, filtros avançados, ações em toolbar |
| `link` | Mínima | Ação que parece link de texto |

## Hierarquia e ordenamento

Botões comunicam prioridade visual. Cada variante sinaliza ao usuário qual ação é mais importante — por isso a combinação e o ordenamento importam tanto quanto a escolha individual.

### Princípio base

> **A ação de menor compromisso fica à esquerda. A ação de maior compromisso fica à direita.**

O olho lê da esquerda para direita. O usuário chega ao botão primário por último — depois de já ter lido todas as opções.

### Frequência por grupo de ações

| Variante | Qtd. por grupo | Posição | Representa |
|----------|---------------|---------|------------|
| `default` | 1 | Direita | "Confirme esta ação" |
| `secondary` | 0–1 | Centro | "Há uma alternativa válida" |
| `outline` | 0–2 | Esquerda | "Sair ou desfazer" |
| `ghost` | Ilimitado | Fora do grupo principal | "Ação contextual que não compete" |

<Note>
`ghost` não faz parte do grupo de ações primário (rodapé de modal, footer de formulário). Ele vive fora desse grupo — em toolbars, cabeçalhos de seção, células de tabela e cards.
</Note>

### Padrões por contexto

| Contexto | Ordenamento (esq → dir) |
|----------|------------------------|
| Modal de confirmação | `Outline: Cancelar` → `Default: Confirmar` |
| Modal destrutivo | `Outline: Cancelar` → `Destructive: Excluir` |
| Formulário com rascunho | `Outline: Cancelar` → `Secondary: Salvar rascunho` → `Default: Publicar` |
| Wizard (step a step) | `Outline: Voltar` → `Default: Próximo` |
| Toolbar de listagem | `Ghost: Filtros avançados` · `Ghost: Exportar` · ... → `Default: Novo item` |
| Ação em linha (tabela/card) | `Ghost: ✏️ Editar` · `Ghost: 📋 Duplicar` (sem grupo formal) |

### Ghost — ações contextuais

O `ghost` opera em uma camada diferente das demais variantes: ele não faz parte de um grupo de decisão, mas age **sobre** um item ou contexto específico.

**Quando usar `ghost`:**

| Contexto | Exemplos |
|----------|----------|
| Ações inline em tabelas | Editar, Duplicar, Visualizar (na linha do registro) |
| Toolbar de página | Filtros avançados, Exportar, Configurar colunas |
| Cabeçalho de seção/card | "Ver todos", "Editar seção" |
| Ações discretas que aparecem no hover | Ícone de editar ao passar o mouse em um campo |

**Regras do `ghost`:**

- ✅ Use quando a ação não compete com um CTA principal próximo.
- ✅ Prefira `size="sm"` ou `size="icon"` em contextos compactos (tabelas, toolbars).
- ✅ Em toolbars, `ghost` pode aparecer múltiplas vezes — mas ações destrutivas sempre usam `destructive`, mesmo em toolbar.
- ❌ Não use `ghost` no grupo de ações de modais ou formulários — use `outline` para cancelar/voltar.
- ❌ Não substitua `ghost` por `outline` em toolbars — o border do `outline` cria ruído visual quando há muitos botões.

## Quando usar / quando NÃO usar

- ✅ Ações: salvar, confirmar, enviar, criar, deletar
- ✅ Abertura de modais, drawers, dialogs
- ❌ Navegação entre páginas — use link (`<a>`) ou Link component
- ❌ Ações secundárias de baixa importância — considere `ghost` ou `link` variant

## Regras

- ✅ Máximo de 1 botão `default` (primário) por grupo de ações.
- ✅ Ordem em grupos de decisão: `outline` (esq) → `secondary` (centro) → `default` (dir).
- ✅ `ghost` é para ações contextuais (toolbars, inline, hover) — não entra no grupo de decisão.
- ✅ Botões destrutivos sempre pedem confirmação via `AlertDialog` antes de executar.
- ✅ Use `size="icon"` com `aria-label` descritivo quando o botão contiver apenas ícone.
- ✅ Estado de loading: desabilite o botão e adicione spinner + texto "Carregando...".
- ❌ Não use `default` variant para ações secundárias.
- ❌ Não coloque o botão `default` à esquerda de outros botões.
- ❌ Não use `outline` em toolbars com múltiplos botões — use `ghost` para reduzir ruído visual.
- ❌ Não use mais de 3 botões no grupo de decisão da mesma linha — consolide em dropdown.

## Exemplos

✅ Correto — cancelar + confirmar (outline → default):
```erb
<%= render RubyUI::Button.new(variant: :outline) { "Cancelar" } %>
<%= render RubyUI::Button.new(variant: :default) { "Salvar" } %>
```

✅ Correto — três níveis em formulário (outline → secondary → default):
```erb
<%= render RubyUI::Button.new(variant: :outline) { "Cancelar" } %>
<%= render RubyUI::Button.new(variant: :secondary) { "Salvar rascunho" } %>
<%= render RubyUI::Button.new(variant: :default) { "Publicar" } %>
```

✅ Correto — ação destrutiva com confirmação:
```erb
<%= render RubyUI::Button.new(variant: :outline) { "Cancelar" } %>
<%= render RubyUI::Button.new(variant: :destructive) { "Excluir fornecedor" } %>
```

✅ Correto — toolbar com ghost + CTA primário:
```erb
<%# Lado esquerdo: ações contextuais discretas %>
<%= render RubyUI::Button.new(variant: :ghost, size: :sm) { "Filtros avançados" } %>
<%= render RubyUI::Button.new(variant: :ghost, size: :sm) { "Exportar" } %>

<%# Lado direito: ação principal %>
<%= render RubyUI::Button.new(variant: :default) { "Novo fornecedor" } %>
```

✅ Correto — ações inline em tabela (ghost):
```erb
<%= render RubyUI::Button.new(variant: :ghost, size: :icon) do %>
  <%= render RubyUI::Icon.new(name: "pencil") %>
<% end %>
```

❌ Errado — outline em toolbar com múltiplos botões:
```erb
<%# Ruído visual desnecessário — use ghost %>
<%= render RubyUI::Button.new(variant: :outline, size: :sm) { "Filtros avançados" } %>
<%= render RubyUI::Button.new(variant: :outline, size: :sm) { "Exportar" } %>
```

❌ Errado — dois botões primários concorrendo:
```erb
<%= render RubyUI::Button.new(variant: :default) { "Salvar" } %>
<%= render RubyUI::Button.new(variant: :default) { "Publicar" } %>
```

❌ Errado — usar Button para navegação:
```erb
<%= render RubyUI::Button.new(href: "/suppliers") { "Ver fornecedores" } %>
```
