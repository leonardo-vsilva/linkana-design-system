|----------|--------|-----|
| `default` | Alta | Ação principal do fluxo — salvar, confirmar, publicar |

## Hierarquia e ordenamento

Botões comunicam prioridade visual. Cada variante sinaliza ao usuário qual ação é mais importante — por isso a combinação e o ordenamento importam tanto quanto a escolha individual.

### Princípio base

> **A ação de menor compromisso fica à esquerda. A ação de maior compromisso fica à direita.**

O olho lê da esquerda para direita. O usuário chega ao botão primário por último — depois de já ter lido todas as opções.

### Frequência por grupo de ações

| Variante | Qtd. por grupo | Posição | Representa |
|----------|---------------|---------|------------|
| `default` | 1 | Direita | "Confirme esta ação" |
| `secondary` | 0–1 | Centro | "Há uma alternativa válida" |
| `outline` | 0–2+ | Esquerda / Toolbar | "Ação secundária que precisa ser descobrível" |
| `ghost` | Ilimitado | Fora do grupo principal | "Ação mínima que não deve chamar atenção" |

<Note>
`outline` cobre dois papéis: ações de saída em grupos de decisão (cancelar, voltar) **e** ações secundárias visíveis em toolbars (editar, filtros avançados, exportar). O `ghost` é reservado para ações verdadeiramente discretas — geralmente `size="icon"` em hover ou contextos muito densos.
</Note>

### Padrões por contexto

| Contexto | Ordenamento (esq → dir) |
|----------|------------------------|
| Modal de confirmação | `Outline: Cancelar` → `Default: Confirmar` |
| Modal destrutivo | `Outline: Cancelar` → `Destructive: Excluir` |
| Formulário com rascunho | `Outline: Cancelar` → `Secondary: Salvar rascunho` → `Default: Publicar` |
| Wizard (step a step) | `Outline: Voltar` → `Default: Próximo` |
| Toolbar de listagem | `Outline: Filtros avançados` · `Outline: Exportar` · ... → `Default: Novo item` |
| Ação em linha (tabela/card) | `Outline: Editar` · `Outline: Duplicar` (sem grupo formal) |
| Ação de hover em campo/linha | `Ghost: ✏️` (ícone, aparece no hover) |

### Outline — ações secundárias visíveis

O `outline` cobre ações que precisam ser descobríveis mas não competem com o CTA principal. Isso inclui tanto ações de saída em grupos de decisão quanto ações funcionais em toolbars e linhas de tabela.

**Quando usar `outline`:**

| Contexto | Exemplos |
|----------|----------|
| Grupo de decisão (modal, formulário) | Cancelar, Voltar, Limpar |
| Toolbar de página | Filtros avançados, Exportar, Configurar colunas |
| Ações inline em tabelas/cards | Editar, Duplicar, Visualizar |
| Cabeçalho de seção | "Editar seção", "Ver todos" |

### Ghost — ações mínimas e contextuais

O `ghost` é reservado para ações que não devem chamar atenção — geralmente ícones que aparecem no hover ou em contextos muito densos onde qualquer borda criaria ruído.

**Quando usar `ghost`:**

| Contexto | Exemplos |
|----------|----------|
| Ícone de ação no hover | Lápis ao passar o mouse em um campo ou linha |
| Contextos extremamente densos | Botões de ação dentro de células pequenas |
| Ações de sistema/UI | Fechar, minimizar, expandir |

**Regras do `ghost`:**

- ✅ Quase sempre `size="icon"` — texto em ghost raramente é necessário.
- ✅ Use quando a ação só precisa aparecer no hover, não o tempo todo.
- ❌ Não use `ghost` para ações que o usuário precisa descobrir sem hover — use `outline`.
- ❌ Não use `ghost` no grupo de decisão de modais ou formulários.

## Ícones

O ícone em um botão deve **carregar significado**, não decoração. Se o usuário precisaria ignorar o ícone para ler o texto de qualquer forma, o ícone está gerando ruído.

### Ícone + texto

Use quando o ícone tem uma **metáfora clara e universal** para a ação — ele reduz o tempo de compreensão e ajuda o usuário a escanear múltiplos botões em linha.

| Ação | Ícone recomendado |
|------|------------------|
| Novo / Adicionar | `plus` |
| Editar | `pencil` |
| Excluir | `trash` |
| Exportar / Download | `download` |
| Upload | `upload` |
| Filtros | `sliders-horizontal` |
| Configurar | `settings` |

Também prefira ícone + texto quando há **múltiplos botões outline em linha** — o ícone diferencia visualmente sem precisar ler cada label.

### Somente texto

Use texto puro quando a ação **não tem metáfora visual óbvia** ou quando o ícone seria puramente decorativo:

- Cancelar, Voltar, Limpar
- Salvar rascunho, Publicar
- Confirmar, Continuar, Próximo

Adicionar um ícone genérico nesses casos (ex.: `check` em "Salvar") não reduz o tempo de leitura — só cria ruído.

### Somente ícone (`size="icon"`)

Use apenas quando **todas** as condições forem verdadeiras:

1. O ícone é **universalmente compreendido** (`✕`, `⋮`, `+`, `←`)
2. O **contexto reforça** o significado — `✕` no canto de um modal sempre significa fechar
3. O espaço é **genuinamente limitado** — célula de tabela, toolbar densa
4. O botão tem **`aria-label`** descritivo para acessibilidade

<Warning>
Nunca use `size="icon"` em CTAs primários. O usuário precisa entender a ação antes de clicar — ícone sozinho em `default` gera hesitação.
</Warning>

### Regras de ícone

- ✅ Ícone à **esquerda** do texto — o olho lê ícone antes do label
- ✅ `size="icon"` sempre com `aria-label`
- ❌ Não adicione ícone se ele não reduz o tempo de compreensão
- ❌ Não misture botões com ícone e sem ícone no mesmo grupo de ações — ou todos têm ou nenhum tem

## Quando usar / quando NÃO usar

- ✅ Ações: salvar, confirmar, enviar, criar, deletar
- ✅ Abertura de modais, drawers, dialogs
- ❌ Navegação entre páginas — use link (`<a>`) ou Link component
- ❌ Ações secundárias de baixa importância — considere `ghost` ou `link` variant

## Regras

- ✅ Máximo de 1 botão `default` (primário) por página.
- ✅ Máximo de 2 botões por linha de ações — use DropdownMenu para o excedente.
- ✅ Ordem em grupos de decisão: `outline` (esq) → `secondary` (centro) → `default` (dir).
- ✅ `outline` para ações secundárias visíveis: cancelar, editar, filtros avançados, exportar.
- ✅ `ghost` para ações mínimas: ícones de hover, ações em contextos muito densos.
- ✅ Botões destrutivos sempre pedem confirmação via `AlertDialog` antes de executar.
- ✅ Use `size="icon"` com `aria-label` descritivo quando o botão contiver apenas ícone.
- ✅ Estado de loading: desabilite o botão e adicione spinner + texto "Carregando...".
- ❌ Não use `default` variant para ações secundárias.
- ❌ Não coloque o botão `default` à esquerda de outros botões.
- ❌ Não use `ghost` para ações que o usuário precisa descobrir sem passar o mouse — use `outline`.
- ❌ Não use mais de 2 botões visíveis na mesma linha — consolide os excedentes em dropdown.

## Hierarquia em linhas de ação

### 1 Default por página

Cada tela deve ter **no máximo um botão `default`** (filled/preto). Ele representa a ação principal e concentra a atenção do usuário. Modais e dialogs isolados podem ter seu próprio `default` sem conflitar com o da página.

### Máximo 2 botões por linha

Quando uma linha de ações teria 3 ou mais botões, agrupe os secundários em um `DropdownMenu` com trigger `•••`. O botão `default` nunca vai para o dropdown — sempre fica visível.

```erb
<%# ✅ Correto — CTA visível + secundária visível + excedentes no dropdown %>
<%= render RubyUI::Button.new(variant: :default) { "Incluir novos" } %>
<%= render RubyUI::Button.new(variant: :outline) { "Exportar" } %>

<%= render RubyUI::DropdownMenu.new do %>
  <%= render RubyUI::DropdownMenuTrigger.new do %>
    <%= render RubyUI::Button.new(variant: :outline, size: :icon) do %>
      <%= lucide_icon "ellipsis", class: "size-4" %>
    <% end %>
  <% end %>
  <%= render RubyUI::DropdownMenuContent.new(align: "end") do %>
    <%= render RubyUI::DropdownMenuItem.new { "Histórico" } %>
    <%= render RubyUI::DropdownMenuSeparator.new %>
    <%= render RubyUI::DropdownMenuItem.new(class: "text-destructive focus:text-destructive") { "Excluir todos" } %>
  <% end %>
<% end %>
```

### Destructive ao lado de Default

Quando um botão destrutivo coexiste na mesma linha com um `default`, **não use o `destructive` filled** (vermelho sólido) — ele compete visualmente com o `default`. Use a variante **outline destrutiva** em vez disso:

```erb
<%# ✅ Correto — destructive como outline quando há um default na mesma linha %>
<%= render RubyUI::Button.new(variant: :default) { "Salvar" } %>
<%= render RubyUI::Button.new(variant: :outline, class: "border-destructive text-destructive hover:bg-destructive/10") { "Excluir" } %>
```

```erb
<%# ❌ Errado — dois botões de destaque competindo na mesma linha %>
<%= render RubyUI::Button.new(variant: :default) { "Salvar" } %>
<%= render RubyUI::Button.new(variant: :destructive) { "Excluir" } %>
```

O `destructive` filled é reservado para quando é a **única ação de destaque** da tela (ex: página de confirmação de exclusão) ou está isolado dentro de um DropdownMenu.
